La sécurité des API

J'ai choisi de réaliser mon sujet de veille sur les API car elles permettent de gagner du temps en programmation, et de voir plus clair dans un programme, pour parler le langage des développeurs et enfin pour éviter les répétitions de code.


Ce qui pour moi était intéressant car je suis appprentie en développement et je pourrais avoir recours aux API dans mon futur professionnel car elles sont aujourd'hui très utilisées et prennent une grande ampleur.
C'est pourquoi je me suis intéressée à étudier leurs architectures et stratégies de sécurisation.


Méthode de veille

Pour la veille j'ai principalement utilisé Google Alert qui est un service qui envoie un courriel ou une alerte lorsqu'une nouvelle page web correspondant aux mots-clés que l'on a choisi, apparaît dans les résultats Google. Ensuite j'ai répertorié toutes ces alertes sur un document afin de trier les articles qui seraient les plus pertinents pour mon sujet.

Une API qu'est-ce que c'est ?

Application Programming Interface est une interface de programmation d’applications, sera un ensemble normalisé de classes, de méthodes, de fonctions et de constantes, qui serviront de façade, par laquelle un logiciel pourra offrir des services à d’autres logiciels.

Stratégie de sécurisation

La sécurité des API consiste à protéger l’intégrité des API, à la fois celles que l’on possède et celles que l’on utilise.
La stratégie de sécurisation d’une API dépend du type de données transférées.
Trafic d’API : attaques systèmes d’identification, vol de données, injection de code malveillant


Basées sur le protocole HTTP et prennent en charge le chiffrement TLS (Transport Layer Security).
Le standard TLS permet de garder une connexion Internet privée et de vérifier que les données échangées entre deux systèmes (un serveur et un serveur, ou un serveur et un client) sont chiffrées et non modifiables

Utilisent des protocoles intégrés connus sous le nom de WS Security (Web Services Security). Ces protocoles définissent un ensemble de règles qui permettent d'ajouter un mécanisme de confidentialité et d'authentification. Les API SOAP sont généralement reconnues pour leurs mesures de sécurité plus complètes

Conclusion

En somme, les API sont de plus en plus utilisées par les développeurs grâce à son efficacité dans l'utilisation de celles-ci. Cependant, elles manipulent beaucoup de données sensibles qui sont convoitées par des personnes malveillantes. L'ampleur que les API connaissent sont également ciblées d'attaques qui perçent leur système de connexion permettant d'avoir accès aux données, code source d'une application ce qui peut être très néfaste pour la e-reputation d'une entreprise par exemple. En effet, de nombreuses techniques de sécurisation ont été mises en place pour pallier à ces attaques. La technique "Zéro-Trust" est celle qui aujourd'hui (nouvelle version "Intelligente) est la plus fiable et ne laisse rien fuiter en bloquant tous les vérrous de ces canaux de connexion.
